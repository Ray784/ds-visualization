How to use?

Our Code Editor
    You can use our code editor to visualise your own data structure(only stacks, queues, trees available) application visualisation (currently only js available) While using our online code editor you don't have to import any module. Just follow the given syntax for stacks, queues and trees accordingly.

Stack

Variables
  1. Variable name: MAX
    • Type: Integer
    • Default value: 6
    • Description: The maximum number of elements allowed in a stack.

  2. Variable name: stacks
    • Type: Array
    • Default value: [  ]
    • Description: list of all stacks created using createStack method.
      • Note: stack in stacks array exists until it is deleted using the deleteDS method.

Note: The values of the variables can be changed by simple assignment ex: MAX=100.

Methods
  1. Method syntax: createStack(arr, show)
    • Parameters:
      1. Name: arr
        • Type: Array
        • Description: list of elements to be used to create a stacks.
        • use [  ] to create empty stack.
      2. Name: show
        • Type: string
        • Description: property of the element to be visualised.
        • leave blank in case of primitive types. ex: createStack([12])
    • Return Type: Stack
    • Description: The function returns a stack with the elements of arr pushed into it.

  2. Method syntax: push(stack, data)
    • Parameters:
      1. Name: stack
        • Type: Stack
          • (probably created using createStack method)
        • Description:The stack to push the element into.
      2. Name: data
        • Type: any
        • Description: element to be pushed into the stack.
    • Return Type: null
    • Description: The function pushes the element into the stack and also visualises the insertion.

  3. Method syntax: pop(stack)
    • Parameters:
      1. Name: stack
        • Type: Stack
          • (probably created using createStack method)
        • Description:The stack to pop the element from.
    • Return Type: any
    • Description: The function pops an element from the stack, visualises the deletion and returns the deleted element.

Attributes
  1. Attribute name: stk_top
    • Type: Integer
    • Default value: -1
    • Description: The top pointer of the stack
  2. Attribute name: length()
    • Return Type: Integer
    • Description: returns the length of stack
  3. Attribute name: isEmpty()
    • Return Type: Boolean
    • Description: returns if the stack is empty or not.
  4. Attribute name: toString()
    • Return Type: String
    • Description: returns a String showing the contents of the stack.
Note: The values of the attribute can be accessed as stack.stk_top



Queue

Variables
  1. Variable name: MAX
    • Type: Integer
    • Default value: 6
    • Description: The maximum number of elements allowed in a queue.

  2. Variable name: queues
    • Type: Array
    • Default value: [  ]
    • Description: list of all queues created using createQueue method.
      • Note: queue in queues array exists until it is deleted using the deleteDS method.

Note: The values of the variables can be changed by simple assignment ex: MAX=100.

Methods
  1. Method syntax: createQueue(arr, show)
    • Parameters:
      1. Name: arr
        • Type: Array
        • Description: list of elements to be used to create a queues.
        • use [  ] to create empty queue.
      2. Name: show
        • Type: string
        • Description: property of the element to be visualised.
        • leave blank in case of primitive types. ex: createQueue([12])
    • Return Type: Queue
    • Description: The function returns a queue with the elements of arr enqueued into it.

  2. Method syntax: enqueue(queue, data)
    • Parameters:
      1. Name: queue
        • Type: Queue
          • (probably created using createQueue method)
        • Description:The queue to push the element into.
      2. Name: data
        • Type: any
        • Description: element to be inserted into the queue.
    • Return Type: null
    • Description: The function enqueues the element into the queue and also visualises the insertion.

  3. Method syntax: dequeue(queue)
    • Parameters:
      1. Name: queue
        • Type: Queue
          • (probably created using createQueue method)
        • Description:The queue to delete the element from.
    • Return Type: any
    • Description: The function deletes an element from the queue, visualises the deletion and returns the deleted element.

Attributes
  1. Attribute name: q_front
    • Type: Integer
    • Default value: -1
    • Description: The front pointer of the queue
  2. Attribute name: q_rear
    • Type: Integer
    • Default value: -1
    • Description: The rear pointer of the queue
  3. Attribute name: length()
    • Return Type: Integer
    • Description: returns the length of queue
  4. Attribute name: isEmpty()
    • Return Type: Boolean
    • Description: returns if the queue is empty or not.
  5. Attribute name: toString()
    • Return Type: String
    • Description: returns a String showing the contents of the queue.
Note: The values of the attribute can be accessed as queue.q_front



Binary Search Tree

Variables
  1. Variable name: trees
    • Type: Array
    • Default value: [  ]
    • Description: list of all binary search trees created using createBST method.
      • Note: a binary search tree in trees array exists until it is deleted using the deleteDS method.

Note: The values of the variables can be changed by simple assignment ex: MAX=100.

Methods
  1. Method syntax: createBST(arr, show)
    • Parameters:
      1. Name: arr
        • Type: Array
        • Description: list of elements to be used to create a binary search tree.
        • use [  ] to create empty BST.
      2. Name: show
        • Type: string
        • Description: property of the element to be visualised.
        • leave blank in case of primitive types. ex: createBST([12])
    • Return Type: Tree
    • Description: The function returns a binary search tree with the elements of arr inserted into it.

  2. Method syntax: insertBST(tree, data)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to insert the element into.
      2. Name: data
        • Type: any
        • Description: element to be inserted into the tree.
    • Return Type: null
    • Description: The function inserts the element into the BST and also visualises the insertion.

  3. Method syntax: deleteBST(tree, data)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to delete the element from.
      2. Name: data
        • Type: any
        • Description: element to be deleted from the BST.
    • Return Type: any
    • Description: The function deletes the specifed element from the BST, visualises the deletion and returns the deleted element.

  4. Method syntax: inorder(tree)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to traverse.
    • Return Type: none
    • Description: The function visualises inorder traversal of the tree.

  5. Method syntax: preorder(tree)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to traverse.
    • Return Type: none
    • Description: The function visualises preorder traversal of the tree.

  6. Method syntax: postorder(tree)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to traverse.
    • Return Type: any
    • Description: The function visualises postorder traversal of the tree.

  7. Method syntax: levelorder(tree)
    • Parameters:
      1. Name: tree
        • Type: Tree
          • (probably created using createBST method)
        • Description:The BST to traverse.
    • Return Type: any
    • Description: The function visualises levelorder traversal of the tree.

Attributes
  1. Attribute name: root
    • Type: Integer
    • Default value: -1
    • Description: The root pointer of the tree
Note: The values of the attribute can be accessed as tree.root

Any data structure

Method
  1. Method syntax: deleteDS(structure)
    • Parameters:
      1. Name: structure
        • Type: Stack/Tree/Queue
          • (probably created using createStack/createBST/createQueue methods)
        • Description: structuree to be deleted from stacks/trees/queues array which stores stacks/trees/queues created using createStack/createBST/createQueue method.
    • Return Type: null
    • Description: The function deletes the specified stack from the stacks array irrespective of its position.
